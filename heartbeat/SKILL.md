# /heartbeat - Fresh Eyes on the Whole System

Reflective scan across the entire project. Surfaces what's active, what's stale, and what's forming — then routes insights where they belong.

---

## When To Use

- Start of a session, especially after time away
- At natural pauses between threads
- When things are accumulating but not connecting
- When the user asks "what are we missing" or "what's alive right now"
- Skip when deep in flow on a specific task

---

## What To Do

Lead with what's most interesting or surprising. Present this as a conversation, not a report.

### 1. Scan

Read across the system in priority order:

**Orient first:**
- **Backlog / scratchpad** — What was the handoff? What's been sitting?
- **Goals** — Are recent sessions moving toward or away from stated direction?
- **Recent sessions** — Last 3-5 session files. What threads were alive? What got said but not acted on?

**Then widen:**
- **AI reflections** — What were previous instances thinking about?
- **Sparks / ideas** — Anything captured but not yet developed?
- **Tasks** — What's stale? What's blocked? What keeps getting deferred?
- **Library / captures** — Recently added items never discussed?
- **Projects** — Any dormant projects that should be active, or vice versa?

Adapt to the system's structure. Scan wherever goals, tasks, captures, and reflections live.

### 2. Reflect Across Four Layers

Each layer builds on the previous one:

- **Pulse** — What's active now? What's new since last heartbeat? What's gone quiet?
- **Patterns** — What connections are forming across sessions? What themes recur? Where is energy concentrated vs scattered?
- **Vital signs** — Movement toward goals? What keeps getting deferred and why? What's conspicuously absent?
- **Emergence** — What's forming that doesn't have a container yet? A spark across 3 sessions might be ready to become something.

**Validation:** If fewer than 3 distinct sources surfaced, expand the scan to additional locations before moving on.

### 3. Route With Intention

| What surfaced | Where it goes |
|---|---|
| New spark or connection | Sparks file, idea log, or backlog |
| Methodology / how you work together | System docs, partnership notes |
| Convention or standard | Rules, guidelines, wherever standards live |
| Topic or domain insight | Knowledge base, reference docs, library |
| Something about the human | Their identity doc or personal notes |
| Something for the AI specifically | AI reflection space |
| Work pattern for a specific project | That project's docs |
| Direction drift or realization | Goals doc or backlog |
| Actionable work | Task doc or backlog |
| Nothing to route | That's fine. Not every heartbeat produces artifacts. |

The session capture preserves the full heartbeat conversation. No separate log needed.

---

## Quick Version

1. Scan: sessions, sparks, backlog, goals, tasks, library, AI reflections, projects
2. Reflect: pulse → patterns → vital signs → emergence
3. Share: conversation, not report — lead with what's interesting
4. Route: insights to where they belong

---

## Skill Check

- Did the scan feel thorough or were obvious corners missed?
- Did the reflection produce genuine insight or just inventory?
- Did routing feel natural or forced?
- Was the timing right — did this interrupt flow, or create clarity?

**If yes** → update this skill now. **If no** → move on.
